# FastAPI Integration

## Introduction

Good news: Redis OM was specifically designed to integrate with FastAPI!

This page covers two integration paths:

1. **[fastapi-redis-sdk](https://github.com/redis/fastapi-redis-sdk)** — Redis' official FastAPI integration for connection-pool management and DI-based HTTP caching. Redis OM ships a small bridge module so your models can share the same managed pool.
2. **Standalone** — bring your own Redis connection and caching layer. Redis OM works with plain `redis-py` and needs no web framework.

## Concepts

### Every Redis OM Model is also a Pydantic model

Every Redis OM model is also a Pydantic model, so you can define a model and then use the model class anywhere that FastAPI expects a Pydantic model.

This means a couple of things:

1. A Redis OM model can be used for request body validation
2. Redis OM models show up in the auto-generated API documentation

### Cache vs. Data

Redis works well as either a durable data store or a cache, but the optimal Redis configuration is often different between these two use cases.

You almost always want to use a Redis instance tuned for caching when you're caching and a separate Redis instance tuned for data durability for storing application state.

Both integration paths below can manage these two uses of Redis within the same application — a dedicated caching instance for HTTP responses and a separate durability-tuned instance for Redis OM models.

## Using the official `fastapi-redis-sdk`

[`fastapi-redis-sdk`](https://github.com/redis/fastapi-redis-sdk) (`pip install fastapi-redis-sdk`) is Redis' official FastAPI integration. It manages async Redis connection pools via a FastAPI lifespan and provides dependency-injection-based HTTP caching (`cache()`, `cache_evict()`, `cache_put()`) plus a lower-level `CacheBackend`. Features include ETag / `304 Not Modified` / `Cache-Control` support, `dependency_overrides`-friendly testing, and Pydantic-validated configuration via `REDIS_*` env vars or a `.env` file.

Redis OM ships a bridge module — `aredis_om.integrations.fastapi_redis_sdk` — that lets your models **reuse the same connection pool** managed by `fastapi-redis-sdk`, so you don't open a second pool to the same instance. The bridge imports `redis_fastapi` lazily, so `aredis_om` keeps working without `fastapi-redis-sdk` installed.

<Note>
The bridge module is **async-only** because `fastapi-redis-sdk` manages `redis.asyncio`
clients. Pair it with `aredis_om` (the async source of truth), not the generated
`redis_om` sync mirror. For sync Redis OM usage, configure `Meta.database` with a
sync `redis.Redis` client directly — see the [standalone example](#standalone-example-without-fastapi-redis-sdk) below.
</Note>

### Install

```sh
pip install pyredis-om fastapi-redis-sdk
# or, with uv:
uv add pyredis-om fastapi-redis-sdk
```

### Share the managed pool (`database_from_app`)

`database_from_app(app)` returns a *callable* suitable for a model's `Meta.database`. Redis OM invokes it lazily the first time the model touches Redis (i.e. after the FastAPI lifespan has started the pool), so it is safe to assign at class-definition time.

```python
from datetime import date
from typing import Optional

from fastapi import FastAPI, HTTPException
from pydantic import EmailStr

from redis_fastapi import (
    FastAPIRedis,
    CacheBackendDep,
    cache,
    cache_evict,
    default_key_builder,
)
from fastapi import Depends

from aredis_om import HashModel, NotFoundError
from aredis_om.integrations.fastapi_redis_sdk import database_from_app

app = FastAPI()

# 1. Let fastapi-redis-sdk own the lifespan and the async connection pool.
#    REDIS_URL (or REDIS_HOST/REDIS_PORT/...) configures it.
FastAPIRedis(app).lifespan().caching()


class Customer(HashModel):
    first_name: str
    last_name: str
    email: EmailStr
    join_date: date
    age: int
    bio: Optional[str] = None

    class Meta:
        # 2. Reuse the same pool fastapi-redis-sdk manages. The callable
        #    resolves on first use, after the lifespan has initialised it.
        database = database_from_app(app)


@app.post("/customer")
async def save_customer(customer: Customer):
    return await customer.save()


@app.get("/customers")
async def list_customers():
    return {"customers": [pk async for pk in Customer.all_pks()]}


@app.get("/customer/{pk}", dependencies=[Depends(cache(ttl=10, eviction_group="customers"))])
async def get_customer(pk: str):
    try:
        return await Customer.get(pk)
    except NotFoundError:
        raise HTTPException(status_code=404, detail="Customer not found")


@app.delete(
    "/customer/{pk}",
    dependencies=[Depends(cache_evict(eviction_group="customers", key_builder=default_key_builder))],
)
async def delete_customer(pk: str):
    await Customer.delete(pk)
    return {"deleted": pk}
```

Because `database_from_app` shares `fastapi-redis-sdk`'s pool, this setup uses a **single** Redis instance for both the durable model data and the HTTP cache. Configure it with one `REDIS_URL`.

### Separate data and cache instances

If you prefer the [Cache vs. Data](#cache-vs-data) split — a durability-tuned instance for Redis OM and a separate cache instance — point the two libraries at different Redis instances:

```python
import os

from fastapi import FastAPI
from redis_fastapi import FastAPIRedis, get_settings

from aredis_om import HashModel, get_redis_connection

app = FastAPI()
# fastapi-redis-sdk reads REDIS_URL for the cache instance.
FastAPIRedis(app).lifespan().caching()


class Customer(HashModel):
    first_name: str

    class Meta:
        # Redis OM reads its own env var and opens an independent pool
        # to a separate durability-tuned instance.
        database = get_redis_connection(url=os.environ["REDIS_OM_URL"])
```

### Honour `fastapi-redis-sdk` settings without sharing a pool

`database_from_fastapi_settings()` builds a Redis OM connection from `fastapi-redis-sdk`'s public `RedisSettings` (env vars prefixed `REDIS_`, or a `.env` file) instead of sharing its pool. Use this when you want both libraries to honour the same configuration but keep their pools isolated, or when the `FastAPI` instance is not available at model-definition time:

```python
from aredis_om import HashModel
from aredis_om.integrations.fastapi_redis_sdk import database_from_fastapi_settings


class Customer(HashModel):
    first_name: str

    class Meta:
        database = database_from_fastapi_settings()
```

The settings are read lazily on first invocation, so environment variables set after import time are picked up.

### Combining caching with model invalidation

The `CacheBackend` dependency is handy when invalidation depends on model state — for example, evicting a dashboard cache after writing through a Redis OM model:

```python
from fastapi import Depends
from redis_fastapi import CacheBackendDep


@app.post("/customer/{pk}/touch")
async def touch_customer(pk: str, cache: CacheBackendDep):
    customer = await Customer.get(pk)
    customer.bio = "updated"
    await customer.save()
    # Cascade-evict any cached views keyed under this group.
    await cache.delete_group("customers")
    return {"ok": True}
```

`CacheBackend` handles JSON serialization, key prefixing, and eviction groups automatically.

## Standalone example (without `fastapi-redis-sdk`)

Redis OM remains fully usable as a standalone library with plain `redis-py` and any FastAPI caching layer you like. The example below uses [`fastapi-cache`](https://github.com/long2ice/fastapi-cache) and manages its own connections — it is the original Redis OM FastAPI example and does **not** require `fastapi-redis-sdk`.

Let's look at an example FastAPI app that uses Redis OM.

**NOTE**: This example code requires dependencies to run. To install the dependencies, first clone the [redis-om-fastapi](https://github.com/redis-developer/redis-om-fastapi) repository from GitHub. Then follow the installation steps later in this document or in that repository's README.md file.

```python
import datetime
from typing import Optional

import redis

from fastapi import FastAPI, HTTPException
from starlette.requests import Request
from starlette.responses import Response

from fastapi_cache import FastAPICache
from fastapi_cache.backends.redis import RedisBackend
from fastapi_cache.decorator import cache

from pydantic import EmailStr

from redis_om import HashModel, NotFoundError
from redis_om import get_redis_connection

# This Redis instance is tuned for durability.
REDIS_DATA_URL = "redis://localhost:6380"

# This Redis instance is tuned for cache performance.
REDIS_CACHE_URL = "redis://localhost:6381"


class Customer(HashModel):
    first_name: str
    last_name: str
    email: EmailStr
    join_date: datetime.date
    age: int
    bio: Optional[str]


app = FastAPI()


@app.post("/customer")
async def save_customer(customer: Customer):
    # We can save the model to Redis by calling `save()`:
    return customer.save()


@app.get("/customers")
async def list_customers(request: Request, response: Response):
    # To retrieve this customer with its primary key, we use `Customer.get()`:
    return {"customers": Customer.all_pks()}


@app.get("/customer/{pk}")
@cache(expire=10)
async def get_customer(pk: str, request: Request, response: Response):
    # To retrieve this customer with its primary key, we use `Customer.get()`:
    try:
        return Customer.get(pk)
    except NotFoundError:
        raise HTTPException(status_code=404, detail="Customer not found")


@app.on_event("startup")
async def startup():
    r = redis.asyncio.from_url(REDIS_CACHE_URL, encoding="utf8",
                          decode_responses=True)
    FastAPICache.init(RedisBackend(r), prefix="fastapi-cache")

    # You can set the Redis OM URL using the REDIS_OM_URL environment
    # variable, or by manually creating the connection using your model's
    # Meta object.
    Customer.Meta.database = get_redis_connection(url=REDIS_DATA_URL,
                                                  decode_responses=True)
```

## Testing the app

You should install the app's dependencies first. This example uses [uv](https://docs.astral.sh/uv/) (the same dependency manager used by redis-om-python itself):

    $ pip install uv
    $ uv sync

Next, start the server:

    $ uv run uvicorn --reload main:app

If you prefer Poetry:

    $ pip install poetry
    $ poetry install
    $ poetry run uvicorn --reload main:app

Then, in another shell, create a customer:
```
    $ curl -X POST  "http://localhost:8000/customer" -H 'Content-Type: application/json' -d '{"first_name":"Andrew","last_name":"Brookins","email":"a@example.com","age":"38","join_date":"2020
-01-02"}'
    {"pk":"01FM2G8EP38AVMH7PMTAJ123TA","first_name":"Andrew","last_name":"Brookins","email":"a@example.com","join_date":"2020-01-02","age":38,"bio":""}
```

Get a copy of the value for "pk," which is the model's primary key, and make another request to get that customer:

    $ curl "http://localhost:8000/customer/01FM2G8EP38AVMH7PMTAJ123TA"
    {"pk":"01FM2G8EP38AVMH7PMTAJ123TA","first_name":"Andrew","last_name":"Brookins","email":"a@example.com","join_date":"2020-01-02","age":38,"bio":""}

You can also get a list of all customer primary keys:

    $ curl "http://localhost:8000/customers"
    {"customers":["01FM2G8EP38AVMH7PMTAJ123TA"]}

## Redis OM with Asyncio

Redis OM is designed to work with asyncio, so you can use Redis OM models asynchronously within FastAPI applications.

The only difference is that you import the Redis OM models from the `aredis_om` module instead of the `redis_om` module.

Here is the previous FastAPI app, but using asyncio-compatible Redis OM code:

```python
import datetime
from typing import Optional

import redis

from fastapi import FastAPI, HTTPException
from starlette.requests import Request
from starlette.responses import Response

from fastapi_cache import FastAPICache
from fastapi_cache.backends.redis import RedisBackend
from fastapi_cache.decorator import cache

from pydantic import EmailStr

from aredis_om import HashModel, NotFoundError  # <- Notice, we import from aredis_om
from aredis_om import get_redis_connection

# This Redis instance is tuned for durability.
REDIS_DATA_URL = "redis://localhost:6380"

# This Redis instance is tuned for cache performance.
REDIS_CACHE_URL = "redis://localhost:6381"


class Customer(HashModel):
    first_name: str
    last_name: str
    email: EmailStr
    join_date: datetime.date
    age: int
    bio: Optional[str]


app = FastAPI()


@app.post("/customer")
async def save_customer(customer: Customer):
    # We can save the model to Redis by calling `save()`:
    return await customer.save()  # <- We use await here


@app.get("/customers")
async def list_customers(request: Request, response: Response):
    # To retrieve this customer with its primary key, we use `Customer.get()`:
    return {"customers": await Customer.all_pks()}  # <- We also use await here


@app.get("/customer/{pk}")
@cache(expire=10)
async def get_customer(pk: str, request: Request, response: Response):
    # To retrieve this customer with its primary key, we use `Customer.get()`:
    try:
        return await Customer.get(pk)  # <- And, finally, one more await!
    except NotFoundError:
        raise HTTPException(status_code=404, detail="Customer not found")


@app.on_event("startup")
async def startup():
    r = redis.asyncio.from_url(REDIS_CACHE_URL, encoding="utf8",
                          decode_responses=True)
    FastAPICache.init(RedisBackend(r), prefix="fastapi-cache")

    # You can set the Redis OM URL using the REDIS_OM_URL environment
    # variable, or by manually creating the connection using your model's
    # Meta object.
    Customer.Meta.database = get_redis_connection(url=REDIS_DATA_URL,
                                                  decode_responses=True)
```

**NOTE:** The modules `redis_om` and `aredis_om` are identical in almost every
way. The only difference is that the `aredis_om` returns coroutines that you must
`await`.
